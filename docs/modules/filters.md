# Filters

## Purpose

Filters let users narrow down issue lists and boards to the work that matters.

## Capabilities

- Status
- Priority
- Labels
- Assignee
- Reporter
- Project
- Milestone
- Team
- Date ranges
- Custom combinations
- Shareable URLs

## Notes

Filters apply in [Views](views.md) like List and Kanban, and complement [Search](search.md). Basic filters (status, priority, labels, assignee, project) are in MVP; shareable URLs are post-MVP. See [MVP scope](../mvp.md).
