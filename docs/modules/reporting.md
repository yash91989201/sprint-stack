# Reporting

## Purpose
Reporting turns issue and project data into team-level metrics so leads can track throughput, progress, and workload distribution.

## Capabilities

- Issue velocity
- Completion rate
- Burndown
- Open vs closed
- Team workload
- Project progress

## Notes

- Reporting builds on [Issues](issues.md), [Status Workflow](status-workflow.md), and [Teams](teams.md) data.
- Team workload metrics rely on [Teams](teams.md) membership and issue assignment.
- Project progress feeds from [Projects](projects.md) status and milestone completion.
- Reporting is **out of MVP scope**; see [MVP](../mvp.md).
