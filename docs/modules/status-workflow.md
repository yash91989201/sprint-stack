# Status Workflow

## Purpose
The status workflow defines the states an issue can move through. It gives teams a shared language for progress and helps surface blockers.

## Capabilities

- Backlog
- Todo
- In Progress
- Review
- QA
- Blocked
- Done
- Archived
- Custom statuses

## Notes

- Status is a core field on every [Issue](./issues.md).
- Status changes appear in the [Activity Timeline](./activity-timeline.md) and can trigger [Notifications](./notifications.md).
- [Views](./views.md) such as Kanban and List are organized around status.
- Custom statuses let a workspace adapt the workflow beyond the default set.
