# Authentication

## Purpose
Authentication provides the identity layer for Sprint Stack. It lets users sign up, sign in, recover accounts, and manage their sessions across the platform.

## Capabilities

- Email & password
- Google & GitHub OAuth
- Magic link login
- Forgot password
- Email verification
- Multi-session support
- Session management
- Invite acceptance
- Account deletion
- Profile onboarding

## Notes

- Invite acceptance ties authentication into the [Workspaces](./workspaces.md) invitation flow.
- Profile onboarding feeds into [User Profiles](./user-profiles.md) and [Settings](./settings.md).
- OAuth and magic link sign-in methods can reduce friction before the workspace invitation step.
