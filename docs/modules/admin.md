# Admin

## Purpose
Admin gives workspace owners and administrators operational oversight: visibility into membership, activity, resource usage, and access control.

## Capabilities

- Workspace analytics
- User management
- Audit logs
- Billing placeholder
- Storage usage
- Invite management

## Notes

- Admin capabilities depend on the workspace [Roles & permissions](workspaces.md) model.
- Audit logs consume data from the [Activity Timeline](activity-timeline.md) and issue lifecycle events.
- User management and invite management overlap with [Workspaces](workspaces.md) member flows.
- Reporting analytics are a separate concern; see [Reporting](reporting.md).
- Admin, advanced audit logs, and billing are **out of MVP scope**; see [MVP](../mvp.md).
